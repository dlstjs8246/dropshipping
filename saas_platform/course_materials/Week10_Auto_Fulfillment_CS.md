## 📖 [이론 파트] 강사 스크립트 및 핵심 자료 (0:10 - 0:30)

### 1. 잘 때도 돌아가는 기계 만들기 (API의 마법)
* **강사 멘트**: "이제 매출이 하루에 1개, 2개 나기 시작할 겁니다. 그런데 매일 아침 쇼피파이를 켜고, 고객 주소를 복사해서, 알리익스프레스에 붙여넣고 결제하고 있으면 여러분은 사장님이 아니라 시급 만 원짜리 알바생이 된 겁니다. 오늘은 Make.com이라는 노코드 툴을 써서 이 단순 반복 노동을 기계에게 넘기겠습니다."

### 2. Trigger(방아쇠)와 Action(행동)
* **강사 멘트**: "자동화는 딱 두 개만 기억하세요. '언제 작동할 것인가?(Trigger)', '무슨 일을 할 것인가?(Action)'. 예컨대 '쇼피파이에 새 주문이 들어오면(Trigger)', '내 슬랙으로 알림을 보내라(Action)' 입니다."

## 💻 [실습 파트] Hands-on 가이드 (0:40 - 2:10)

> 💡 **강사용 주의사항**: 초보자가 Make 모듈을 처음부터 연결하면 100% 에러를 내고 포기합니다. 반드시 `04_Make_Automation_Guide.md`에 정리된 **사전 제작된 Blueprint(.json) 파일**을 나눠주고 'Import' 기능으로 불러오게 하세요.

### Step 1. 신규 주문 슬랙 알림 봇 (30분)
1. Make.com 가입 후 새 시나리오를 만들고 강사가 준 JSON 파일을 Import 합니다.
2. **연결 (Connection)**: 쇼피파이 모듈을 클릭하고 본인의 스토어 계정 권한을 승인합니다.
3. 슬랙(또는 지메일) 모듈을 클릭해 알림을 받을 채널을 연결합니다.
4. 하단의 'Run Once'를 누른 뒤 쇼피파이에서 가짜 주문을 하나 넣고 슬랙으로 주문 데이터(상품명, 주소)가 날아오는지 테스트합니다.

### Step 2. 자동 리뷰 요청 이메일 세팅 (60분)
1. 두 번째 Blueprint를 Import 합니다. (트리거: 주문 상태가 '배송 완료'로 변경됨)
2. `Sleep` 모듈을 확인합니다. "배송되자마자 리뷰 써달라고 하면 고객이 싫어합니다. 여기서 3일 대기(딜레이)를 줍니다."
3. `Gmail` 모듈을 클릭하고 이메일 템플릿을 수정합니다.
   * *예시*: "안녕! 물건은 잘 받았어? 사진 리뷰를 남겨주면 다음 구매 때 쓸 20% 할인 쿠폰을 바로 쏴줄게!"
4. 시나리오를 'ON'으로 켜서 상시 가동 상태로 만듭니다.

## 🛡️ 예상 Q&A 방어 시나리오

**Q: Make.com 쓰면 돈이 많이 드나요?**
> A: "무료 버전으로도 한 달에 1,000번의 작동(Operation)이 가능합니다. 초보 시절에는 이걸로 차고 넘칩니다. 주문량이 폭발해서 무료 한도를 넘기면 그때 월 $9짜리 유료 플랜을 쓰시면 됩니다. 그땐 이미 돈을 넉넉히 벌고 있을 테니까요."

**Q: 연동 중에 자꾸 401 Unauthorized 에러가 나요!**
> A: "비밀번호나 API 키 권한이 잘못되었을 때 나는 에러입니다. 연결(Connection)을 아예 지웠다가 다시 처음부터 스토어 로그인을 해보세요." (※ 여기서 수강생들이 느끼는 짜증과 고통을 잘 기록해 둡니다. 나중에 14주차 SaaS 런칭 시 "이 에러 나는 거 짜증나셨죠? 제 SaaS는 버튼 하나면 됩니다"라고 세일즈할 강력한 무기가 됩니다.)
